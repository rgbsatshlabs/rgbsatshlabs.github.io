What is Stash

Understanding the importance of Stash and the risks of data loss

Data Loss and Asset Freezing

Asset Records Cannot Be Recovered

Stash stores all transaction records and state information related to a user's RGB assets. Once lost, users will be unable to prove which assets they once owned and the asset transfer process. For example, after Alice receives RGB assets transferred from Bob, the related data is stored in Alice's Stash. If the Stash is lost, Alice cannot prove to others that she received these assets.

Assets May Be Permanently Frozen

Due to the non-global broadcast nature of the RGB protocol, asset status and transfers are only recorded in the local Stash of the trading parties. If the recipient's Stash is lost and there is no backup, these assets may become unusable or untransferable, effectively being permanently frozen, making transaction verification difficult.

Unable to Verify Transaction Validity

In the RGB protocol, verifying whether a transaction is valid requires viewing the historical transfer records of the assets corresponding to that transaction. If the Stash is lost, users will be unable to provide complete transaction history data to prove the legitimacy of current transactions. For example, when Bob wants to transfer RGB assets to Alice, he needs to select all historical transfer records related to the asset from the local Stash, together with the newly generated transfer record, and submit them to Alice for verification. If the Stash is lost, Bob cannot provide this data, and Alice cannot verify the transaction's validity.

Impact on Network Transaction Trust

Due to the inability to verify transaction validity, the transaction trust mechanism of the entire RGB network will be impacted. Other users may become concerned about increased transaction risks due to counterparty Stash data loss, which negatively affects the trading activity of the entire network.

Network Isolation Effect Intensified

Data Isolation Problem Highlighted

The RGB protocol inherently has a data isolation problem, where data between different clients cannot be effectively synchronized and shared. When Stash is lost, this isolation effect is further intensified, as users who have lost data will be unable to effectively interact with other users in the network for data exchange and asset transfers.

Network Integrity Compromised

The intensification of the data isolation effect will compromise the integrity of the RGB network, making asset transfers and transaction coordination between users more difficult. This not only affects user experience but also limits the scale expansion and ecosystem development of the RGB network.

Our Vision

Decentralized Stash Hosting Network

We are building a decentralized Stash hosting network for the RGB protocol to address critical issues in the current ecosystem.

Why we need a Stash hosting network:

  • Currently, assets issued by various projects are hosted on their own servers. Even if some projects realize the need for distributed storage, the risks remain significant.
  • On-chain transparency and multi-party cross-verification. Transparency does not mean insecurity. Users can sign authorizations and then publicly share stash with authorized individuals/projects to view transaction history.

Extended Asset Transaction Records

Asset transaction records are no longer limited to the trading parties. You can share Stash with third-party service providers, such as for generating monthly payment statements or on-chain transaction explorers, to obtain a better trading experience.

Multi-Party Data Recording and Traceability

RWA (Real World Assets), invoices, financial reports, certificates, and other data recording and traceability will require multi-party participation for reconciliation, which will greatly enhance data transparency and credibility.

Privacy, Security & Reliability

Enhanced Security and Reliability

With the introduction of a decentralized Stash hosting network, you no longer need to worry about local wallet crashes or issuers losing your assets. This provides a more solid guarantee for the secure storage and management of assets.

  • Decentralized Storage: Assets are stored across multiple nodes, eliminating single points of failure.
  • Privacy Protection: User signature authorization ensures that only authorized parties can access transaction history.
  • Data Integrity: Multi-party cross-verification ensures the authenticity and reliability of transaction records.
  • Backup & Recovery: Automatic backup mechanisms prevent data loss even in extreme scenarios.

Technical Solution

Stash Hosting Architecture

Our technical solution provides a comprehensive infrastructure for RGB protocol asset management:

  • Decentralized Storage Network: Integration with AR (Arweave) and IPFS for distributed storage of Stash data.
  • Wallet Integration: Support for multiple wallet providers with easy SDK integration.
  • Infrastructure Services: Comprehensive SDK and API services for developers to build on top of RGB protocol.
  • Transaction Explorer: Built-in RGB transaction browser for viewing and analyzing transaction history.

Future: Self-Built Storage Network

Our long-term vision includes building our own storage network to achieve fully autonomous decentralized Stash hosting, providing a unified privacy data sharing platform.

Revenue Sources

RGBS.FUN DEX Trading Fees

Our primary revenue stream comes from transaction fees on the RGBS.FUN DEX platform. We charge a fee based on a percentage of each transaction volume, ensuring sustainable operation while maintaining competitive rates for users.

RGB Transaction Explorer API

The RGB Transaction Explorer provides public API access for developers and service providers. We operate on a pay-per-use model, charging based on the number of API calls made, allowing flexible and scalable access to transaction data and analytics.

Stash Storage Hosting

Users can opt for our decentralized Stash storage and hosting services for enhanced security and reliability. We charge based on individual wallet addresses, providing transparent and predictable pricing for asset storage and management.
